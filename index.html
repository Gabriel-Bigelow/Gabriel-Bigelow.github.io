Four Square

Four Square is a multiplayer game based on the playground game of the same name.

4 players are each assigned to their own square, adjacent to the other 3 players' respective square. The player in the first square hits, or "serves", the ball into an opposing player's square, where the ball must first bounce a single time in the opposing player's square before that player can then hit, or "return", the ball into an opposing player's square.

Players in the arena are free to move anywhere on the screen. The ball is constrained to the arena by walls that the players can also bounce the ball off of. To reach a fast moving ball or to deliver more power on a shot, players can use the sprint ability that drains during use and refills when not used after a short period of time.

    A player is out if they:
  • hit the ball before it has bounced in their square.
  • fail to return the ball before it bounces a second time.
  • hit the ball more than once when serving or returning a hit.
  • return the ball, but it does not land in, or on the edge of an opposing player's square.

When a player is out, they are removed from their specific square and sent to the 4th square, and each player behind them rotates into the square that is their current square minus 1. (If more than 4 players, the next player in the queue is removed from the queue and rotated into the 4th square, and the "out" player is now last in the queue.)

    Controls:
  • Player 1: (movement) arrow keys, (sprint) shift key
  • Player 2: (movement) WASD keys, (sprint) C key
  • Player 3: (movement) TFGH keys, (sprint) N key
  • Player 4: (movement) IJKL keys, (sprint) . key

Quote API

Quote API is a simple demonstration of the understanding of full stack development.

Quote API was created as part of my coursework in the premium Full Stack Development course on Codecademy

Utilizing Node.js and the Express JavaScript framework, a local server is hosted that allows for GET and POST requests to be made from the user interface.

A user can make a GET request for the entire database by clicking the "Fetch All Quotes" button, or return a single quote by clicking the "Fetch Random Quote". A user can also retrieve all quotes by a specific author by inputting a specific author into the text box and clicking "Fetch by Author".

If a user wants to add a quote to the database, they can click "Add a New Quote" at the top of the page. They are then prompted to input a quote and author for the quote. When both fields are filled out, the user can click "Submit Your Quote" to see a success message and return to the home screen to view all quotes and verify their quote was added to the database.
